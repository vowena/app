# Support

## Where to start

- Read [README.md](README.md) for local setup.
- Read [CONTRIBUTING.md](CONTRIBUTING.md) before opening a pull request.
- Read [SECURITY.md](SECURITY.md) if your report involves a vulnerability.

## When to open an issue

Open an issue when you have:

- A reproducible bug
- A scoped feature proposal
- A documentation gap tied to this repository

Use the issue templates so maintainers get the context they need quickly.

## Questions and early ideas

If discussions are enabled for this repository, use them for architecture questions and early product ideas. If they are not enabled yet, open an issue with enough context for maintainers to route it properly.
